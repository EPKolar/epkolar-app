## FIX 4 — Werkzeug-Kalib Null-Guard (Sebastian-Befund: 298 statt ~4)

**Status:** App-Code hat **bereits Null-Guards**:
- Z5283 `if(!w.naechsteKalib) return false`
- Z18007 `w.naechsteKalib && w.naechsteKalib<=td2()`
- Z18032 `if(w.naechsteKalib && w.naechsteKalib<=today)cnt.kalib_faellig++`

**Sebastian's Live 298 deutet auf DB-Daten-Problem:**
- Empty-String `""` wird durch `&&` filter raus ✓
- ABER: `"0000-00-00"` oder `"1970-01-01"` ist truthy + lexikographisch ≤ td2() → würde **fälschlich überfällig**

**Härteres Fix (additiv, kein Bruch):**
```js
const _isValidKalibDate = (s) => /^\d{4}-\d{2}-\d{2}$/.test(s) && s !== "0000-00-00" && s > "1980-01-01";
const kalibFaellig = werkzeuge.filter(w => _isValidKalibDate(w.naechsteKalib) && w.naechsteKalib <= td2()).length;
```

**Plus optional Kategorie-Filter** (nur kalibrierpflichtige Werkzeuge):
```js
const _kalibPflichtKats = ['mess','maschine','elektro','sicherheit'];
const istKalibPflichtig = (w) => _kalibPflichtKats.some(k => (w.kat||"").toLowerCase().includes(k));
```
