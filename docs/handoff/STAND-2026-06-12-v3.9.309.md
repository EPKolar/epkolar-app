## Heute (v3.9.309)
### Kachel-Klick scrollt zu Schein-Liste (`ArbeitsscheinView`)
User-Bug: Monteur klickt Status-Kachel (z.B. „Offen alle"), Filter wird gesetzt + `sub="liste"` umgestellt — aber Liste ist außerhalb des Viewports (Kacheln sind oben, Liste unten). User denkt „macht nichts".

**Fix:** Neuer Helper `_scrollToScheinListe(status)` (~Z.6253) — setzt Filter, schaltet auf Liste-Tab, dann via 80-ms-`setTimeout` + `scrollIntoView({behavior:"smooth",block:"start"})` zur Liste. Ref `_scheinListRef` (~Z.6253) auf dem Liste-Container (~Z.6877). Alle 4 Kacheln (Gesamt / Offen / Einzel-Status-Map / Fertig) verwenden den Helper.

Pattern wiederverwendbar für künftige Kachel-Klicks in anderen Views (Projekte, Mängel, Termine).
